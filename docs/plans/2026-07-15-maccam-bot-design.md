# MacCam Bot Design

MacCam Bot is a small, foreground-only macOS process that lets one Telegram account request a current webcam photo or a short silent video. It uses Telegram long polling, so the Mac does not expose an inbound port. The bot token and allowed Telegram user ID come from a local `.env` file that Git ignores. Authorization requires both the configured user ID and a private Telegram chat; group and channel requests are rejected. The application has no launch agent, hidden process, motion detection, continuous recording, microphone recording, or cloud storage beyond the media message the user explicitly requests.

The runtime is split into configuration, authorization, webcam capture, and Telegram handlers. A capture component invokes the installed Homebrew FFmpeg binary with macOS AVFoundation. Photos warm up the camera briefly before selecting a frame. Videos are H.264 MP4 files capped to a configurable duration between one and ten seconds, with ten seconds as the default. A single asynchronous lock prevents overlapping camera requests. Each request uses a private temporary directory, and the file is removed immediately after Telegram finishes or reports an error.

Commands are `/start`, `/status`, `/photo`, and `/video`. Status reports that the bot is online and whether capture is currently busy, without exposing secrets or local paths. Capture errors are logged locally and returned as concise user-facing messages. Startup validates configuration and FFmpeg availability before polling. Tests cover configuration boundaries, authorization, FFmpeg command construction, cleanup, concurrency, and handler behavior without accessing the real webcam or Telegram network. The README explains BotFather setup, finding the user ID, selecting the AVFoundation camera index, macOS permission behavior, foreground startup, and safe shutdown.
